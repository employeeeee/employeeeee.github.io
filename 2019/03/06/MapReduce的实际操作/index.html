MapReduce的高级特性

  1. 序列化

    • Java的序列化
      serializable 接口

    • MapReduce 的序列化: 核心接口 : Writeble

      如果一个类实现了Writable的接口 该类的对象可以作为key和value

读取员工的数据 生成员工对象 直接输出到HDFS
使用MapReduce序列化 重写求部门工资总额的例子

  2. 排序

    • 规则: 按照key2排序

    • 基本数据类型

      • 数字 默认:升序

        可以改变默认的排序规则(创建自己的比较器)

      • 字符串 默认:字典顺序

    • 对象

      • sql 的排序
        order by后面 + 列名 表达式 别名 序号
        oracle 排序的结果 是临时表的数据 不是原来的表
      • 员工对象的排序 接口WritableComparable
        • 一个列排序
        • 多个列排序 有几个条件就写几个条件 把等号写到最后一个条件上.
          (前提) 该对象必须是Key2
          实现了hadoop序列化接口
对象是可排序的 类似 Java的对象排序 comparable

  3. 分区

    • 什么是分区? partition
      结合关系型数据库Oracle
    • MR的分区: 根据Map的输出<key2,value2>进行分区
    • 默认的情况下 MR只有一个分区 (一个分区就是一个文件)
    • 自定义分区 : 按照员工的部门号进行分区

  4. 合并 Combiner

    • 是一种特殊的Reducer
    • 合并实在Map端执行一次合并 用于减少Mapper输出到Redeucer的数据量 可以提高效率

MapReduce核心: shuffle(洗牌)

  1. Hadoop 3.x 以前: 会有数据落地(产生I/O操作)
